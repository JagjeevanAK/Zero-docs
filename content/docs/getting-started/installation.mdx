## Environment Configuration

### 1. Environment Variables

Create and configure your environment file:

```bash title="Copy .env.example"
cp .env.example .env.local
```

### 2. Required Environment Variables

```bash title="Required .env Variables"
# Application
NODE_ENV=development
PORT=3000
APP_URL=http://localhost:3000

# Database
DATABASE_URL=sqlite:./data/db.sqlite
# Or for PostgreSQL
# DATABASE_URL=postgresql://user:password@localhost:5432/zero

# Authentication
BETTER_AUTH_SECRET=your-super-secret-key-here
BETTER_AUTH_URL=http://localhost:3000

# Email Provider Configuration
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password

# OAuth Providers (Optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

# Storage (Optional)
CLOUDFLARE_R2_ACCESS_KEY_ID=your-access-key
CLOUDFLARE_R2_SECRET_ACCESS_KEY=your-secret-key
CLOUDFLARE_R2_BUCKET=your-bucket-name

# AI Features (Optional)
OPENAI_API_KEY=your-openai-api-key
```

### 3. Database Setup Options

#### Option A: SQLite (Default)
```bash title="SQLite Setup"
# No additional setup required
# Database file will be created automatically
```

#### Option B: PostgreSQL
```bash title="PostgreSQL Setup"
# Install PostgreSQL
brew install postgresql  # macOS
sudo apt install postgresql postgresql-contrib  # Ubuntu

# Start PostgreSQL service
brew services start postgresql  # macOS
sudo systemctl start postgresql  # Ubuntu

# Create database
createdb zero_dev
```

#### Option C: Docker Database
```bash title="Docker Database Setup"
# Start database with Docker Compose
docker-compose -f docker-compose.db.yaml up -d
```

## Installation Steps

### 1. Install Dependencies

```bash title="Install Dependencies"
cd zero
pnpm install
```

### 2. Generate Database Schema

```bash title="Generate DB Schema"
pnpm db:generate
```

### 3. Run Database Migrations

```bash title="Run DB Migrations"
pnpm db:migrate
```

### 4. Seed Database (Optional)

```bash title="Seed Database"
pnpm db:seed
```

### 5. Start Development Server

```bash title="Start Development Server"
pnpm dev
```

## Platform-Specific Instructions

### macOS

```bash title="macOS Installation"
# Install Homebrew (if not installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js via Homebrew
brew install node

# Install pnpm
brew install pnpm

# Continue with standard installation
```

### Windows

```powershell title="Windows Installation"
# Install Node.js from nodejs.org or via Chocolatey
choco install nodejs

# Install pnpm
npm install -g pnpm

# Continue with standard installation
```

### Linux (Ubuntu/Debian)

```bash title="Linux Installation"
# Install Node.js via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install pnpm
curl -fsSL https://get.pnpm.io/install.sh | sh
```

## Docker Installation

### Using Docker Compose (Recommended)

```bash title="Terminal"
# Development environment
docker-compose up -d

# Production environment
docker-compose -f docker-compose.prod.yaml up -d
```

### Manual Docker Setup

```bash title="Terminal"
# Build the image
docker build -t zero-email .

# Run the container
docker run -p 3000:3000 -e NODE_ENV=production zero-email
```

## Troubleshooting

### Common Installation Issues

**Node.js version mismatch**
```bash title="Terminal"
# Use Node Version Manager (nvm)
nvm install 20
nvm use 20
```

**Permission errors on Linux/macOS**
```bash title="Terminal"
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
```

**Port already in use**
```bash title=".env"
PORT=3001
```

**Database connection errors**
- Verify DATABASE_URL in .env
- Ensure database server is running
- Check firewall settings
